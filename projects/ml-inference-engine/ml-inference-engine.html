Architecture

How it works

A four-layer stack from model loading to tensor execution—entirely inside the browser sandbox.

01 / 04
📦
ONNX Model Loader
Parses ONNX protobuf graphs and resolves operator schemas. Supports opset 11–20 with automatic fallback to CPU kernels for unsupported ops.
ONNX · Protobuf
02 / 04
⚡
Quantization Engine
INT4/INT8 post-training quantization with symmetric and asymmetric modes. Calibrates per-channel or per-tensor with < 1% accuracy loss on most vision and NLP tasks.
INT4 · INT8 · FP16
03 / 04
🦀
Rust WASM Runtime
Compiled from safe Rust to WASM via wasm-pack. SIMD-accelerated tensor ops using WebAssembly SIMD extensions. Zero-copy buffer sharing with JS via SharedArrayBuffer.
Rust · wasm-bindgen
04 / 04
🖥️
WebGPU Backend
Falls back gracefully from WebGPU → WebGL2 → WASM CPU. GPU execution pipelines are compiled once and cached. Matrix multiply kernels beat cuBLAS for batch-1 inference.
WebGPU · WGSL

src/runtime/session.rs 
// Load & run an ONNX model with WebGPU acceleration
use mlie::{Session, Backend, QuantMode};

pub async fn run_inference(model_bytes: &[u8]) -> Result<Tensor> {
    let session = Session::builder()
        .backend(Backend::WebGpu)          // falls back to WASM
        .quant_mode(QuantMode::Int8)       // 4x smaller weights
        .cache_kernels(true)               // compile once
        .build_from_bytes(model_bytes)await?;

    let input = Tensor::from_slice(
        &[1.0_f32, 0.5, -0.3, 0.8],
        &[1, 4]           // [batch, features]
    );

    session.run(&[("input", &input)])await
}

Install

Get started in minutes

Three ways to use MLIE — choose the one that fits your workflow.

Terminal 
$ npm install @mlie/core
added 1 package in 0.4s

$ cat > inference.mjs <<'EOF'
import { Session } from "@mlie/core";

const session = await Session.fromURL("./model.onnx");
const result  = await session.run({ input: Float32Array.from([...]) });
console.log(result.output);
EOF

$ node --experimental-wasm-modules inference.mjs
✓ WebGPU backend initialized
✓ Model loaded in 34ms
Float32Array [0.92, 0.05, 0.03]
$

Features

Everything you need

Built from the ground up for the browser — no Node.js, no backend, no concessions.

🔒
Privacy-first
All inference happens on-device. User data never leaves the browser. No telemetry, no cloud calls.
🎯
Quantization-aware
INT4 and INT8 quantization baked in. Shrink a 50MB model to 12MB with minimal accuracy trade-off.
🚀
WebGPU Accelerated
Custom WGSL compute shaders for matrix multiply and conv ops. Automatic fallback to WebGL2 or WASM.
🔄
Streaming inference
Async generator API for token-by-token output. Ideal for language models and real-time use cases.
🧩
ONNX native
Supports ONNX opset 11–20. Export from PyTorch, Keras, or Hugging Face and run without conversion.
⚙️
Embeddable
Works in Service Workers, Web Workers, and Electron. Designed for offline-first progressive web apps.
